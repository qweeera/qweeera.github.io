float型变量为什么只提供7位有效数据

Posted on 2020-05-02

float型变量为什么只提供7位有效数据

首先申明一下，这个7位是指整数部分和小数部分一共7位；
c/c++中，一个float 变量占据一个字（32 bits）; double (64bits)；
遵照IEEE制定的浮点数表示法：
| 类型 | 符号位 | 阶码 | 尾数 | 长度 |
| —— | —— | —- | —- | —- |
| float | 1 | 8 | 23 | 32 |
| double | 1 | 11 | 52 | 64 |

计算机内都是以二进制来表达；
比如 9.125可以表达为
9 的二进制 1001
0.125 的二进制表示为 0.001
所以9.125 二进制表示为 1001.001

1 * 2^3 + 0 * 2^2 + 0 * 2^1 + 1* 2^0 + 0 * 2^-1 + 0 * 2^-2 + 1 * 2^-3
任何一个数都可以通过泰勒多项式来展开表达。

科学计数法为1.001001 * 2 ^ 3

在计算机中，任何一个数都可以表示为1.xxxx*2^n

其中xxxx部分就是他的尾数， n表示指数部分；
因为最高位这个1在表示任何一个数的形式时这里都是1，所以存储时实际上不保存这一位，这使得float的23bits的尾数可以表示24bits的精度， double中52bits的尾数可以表示53bits的精度。

通过上述计算机存储浮点数的方式，如何确定浮点数的精确度呢？
我们以float的极限情况而定，尾数占23位，最大1.111_1111_1111_1111_1111_1111
换算成十进制是 16777215； 也就是说 最大只能表达十进制8位

1/假定指数值大于等于23 则表达式表达都是整数，且精度达到8位
0xFFFFFF 16777215
0xFFFFFF 16777215.1
0x10000 16777216

2/ 如果指数小于23，则必定有一位小数，此时最后一位式不确定位， 也就是说，有效位只有7位
8388607.5
8388607.0
8388607.1 最后一位的1 无法正确表达出来， 只能表达出8388607.0
