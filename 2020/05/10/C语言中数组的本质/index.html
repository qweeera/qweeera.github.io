数组和指针

发表于 2020-05-10 阅读次数：

本文目标：使编程者能够将数组运算与对应指针运算融汇贯通，能够自如切换。

本文需要弄清楚的几个问题：

1、数组的本质是什么

2、 为什么数组容易和指针弄混

事情要从一个数组名开始说起

int b[10];
声明一个数组， 可以看到b[0]、b[1]、b[2]等这些数组元素的数据类型都是一个整型， 那b的类型是什么呢？
注意，此时数组名b有两种属性；一种是数组属性， sizeof(b) = 10 * 4; 另一种是指针常量属性，调用b指向的是数组第一个元素的地址， 这个指针常量的类型取决于数组第一个元素的类型

只有数组名被sizeof()调用的时候，才会显示出数组属性；

其他时候调用都是指针常量属性。

关于数组的本质

1、C语言中只有一维数组，而且数组的大小必须在编译器就作为一个常数确定下来。

2、对于一个数组，我们只能做两件事：

  • 确定该数组的大小；
  • 获得该数组下标为0的元素的指针；

C语言中数组的元素可以是任何类型的对象，当然也可能是另外一个数组，这也就是多维数组的由来。

数组到指针的转换

这里需要用到一个很常见但是又极易被忽略掉的符号 [ ], 这个符号由于一直和数组一起使用，无论是数组的初始化还是数组的访问或者赋值，导致编程人员以为这就是组成数组一部分；

其实不然，这个符号是数组与指针之间的桥梁。

在数组的本质中我们说过，数组只能做两件事之一是获得该数组下标为0的元素的指针。

我们将array[3] 拆分成两部分 array + [3]

数组array在被调用的时候提供了一个下标为0的元素的地址，后面的[3]是基于这个地址做偏移；

array[3] 等价于 *(array + 3); 这里可以看到 array 提供的是一个下标为0的元素的地址指针，通过初始化的定义可知，这个指针是int型。

通过[ ]符号，我们可以进行整个数组各个元素的访问、赋值。

一维数组的理解难度不大，接着我们来看下多维数组；

那么此时*(array + 3)又是什么意思呢？

先前已经反复强调，array是获得该数组下标为0的元素的指针； 此时array[0]里面装的实际是一个长度为4的一维数组，那么array实际就是一个指向长度为4的数组的指针。

因此*（array+3）等价于 array[3]; 返回的是一个长度为4的数组；接着，重点来了，当我们得到一个数组时，就可以获取到该数组第一个元素的地址，array的第一个元素是一个长度为4的数组，因此通过array[x] 我们可以获取一个长度为4的数组的首地址，通过array[x][y] 可以访问二维数组里面的任意数据；

通过上述讲解，可以知道除了sizeof调用数组外，其他时候数组都是指针常量属性。接下来着重介绍这一属性。

数组的指针常量属性

常量和变量的概念

这里对常量和变量的概念做一下详细的解释，比如说int a = 5; 其中a是一个整型变量，那5这个立即数就是一个常量，又比如 #define NUM 5; 这个num就是常量；变量在声明的时候，是有内存空间的，可以被赋值；但是常量声明的时候是没有内存空间的，因此常量只能作为右值，不能被赋值。

数组和指针的初始化

声明一个数组时，编译器将根据声明所指定的元素数量为数组保留内存空间，然后再创建数组名，它的值是一个常量，指向这段空间的起始地址。声明一个指针变量时，编译器只为指针本身保留内存空间，它并不为任何整型值分配内存空间。

明白上述声明之后，可以确定表达式*a 是完全合法的，但是表达式 *b 却是非法的，因为*b将访问内存中某个不确定的位置。
b++ 可以通过编译，但是a++ 不行； 因为b是变量可以被赋值，a是常量不能被赋值。

数组名始终是一个指针常量吗？

不是所有的场合数组名都可以用指针常量来表示的，其中有两个特例：

1/ 当数组名作为sizeof操作符的操作数时

当数组名做为sizeof操作数时，sizeof返回整个数组的内存长度，而不是指向数组的指针的长度。

2/ 当数组名作为单目操作符&的操作数时

当用&操作一个数组名时，返回的是一个指向数组的指针，而不是指向指针常量的指针。
为什么会有这样的特例呢？
首先我们想想，sizeof 或者& 操作一个常量，这个本来就不合理；此时的数组名真正有了数组的特质。
通过sizeof获取整个数组的长度，通过&返回的是指向数组指针。指向数组的指针该如何表达呢？我们带着这个疑问继续往下看。

如果将数组名作为函数参数呢？

现在已经知道在大多数情况下数组名是一个指向第一个元素的常量指针，那么作为函数参数就是该指针的一份拷贝。由于是拷贝，就一定有赋值。因此，函数形参的实质是一个指针变量，它被数组名（指针常量）赋值。
因为是一个指针变量，因此我们可以对该指针进行++, — 操作，由于该指针是一份拷贝，因此调用程序的实参不受影响（但可能影响它所指向的内容）。

为了让新手更容易上手，编译器也支持数组形式的函数形参，这两个函数原型是等价的（但也仅仅是在作为函数形参的时候一致！！）；哪一个更准确一点呢？ 由于是指针变量，很明显用上一种表达更准确，因为实际的操作是将一个指针常量（实参）赋值给一个指针变量（形参）；

字符数组的初始化

char message[] = { 'H', 'e', 'l', 'l', 'o', 0 };
char message[] = "hello";
两种表达方式等价， 尽管第二个看上去像是一个字符串常量，实际上并不是。它只是前例初始化列表的另一种写法。如果分辨字符串常量和这种初始化列表呢？
实际上是根据他们所处的上下文环境进行区分的，当用于初始化一个字符数组时，它就是一个初始化列表，在其他任何地方，它都表示一个字符串常量。
char message[] = "hello";
char *message2 = "hello";
这两个看上去很像，但他们具有不同的涵义。前者时初始化一个字符数组的元素，后者则是一个真正的字符串常量。这个指针变量被初始化为指向这个字符串常量的存储位置，如下图所示：

二维数组名

一维数组名是一个指针常量，它的类型是”指向元素类型的指针”，它指向数组的第一个元素。多维数组的第一个元素实际上是另一个数组,也就是说，多维数组名也是一个指针常量，他的类型是指向下一维数组的指针。比如说三维数组名，它就是指向二维数组的的常量指针；
int matrix[3][10]
它可以看作是一个一维数组，包含3个元素，每个元素的类型是包含10个整型元素的数组。
matrix 这个数组名的值是指向数组第一个元素的指针， 所以matrix是一个指向”一个包含十个整型元素的数组”的指针，简而言之，matrix是一个指向数组的指针！

如何声明一个指向整型数组的指针呢？
int (*p)[10];
该声明没有明确告诉你P是什么，但是对他进行推论可知是一个指向整型数组的指针。
int (*p)[10] = matrix;
